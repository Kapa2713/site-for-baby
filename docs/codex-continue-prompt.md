# Продолжение задачи для Codex

Я посмотрел первый MVP-коммит `Добавить MVP сайта для гендер-пати`.

В целом направление правильное:

```text
статический сайт
→ Google Apps Script
→ Google Таблица
```

Первый MVP должен оставаться бесплатным или максимально близким к бесплатному варианту.

Google Таблицу, Apps Script-проект, публикацию Web App и GitHub Pages я буду настраивать вручную вместе с ChatGPT. Поэтому сейчас не нужно создавать настоящую Google Таблицу, не нужно просить реальные ссылки, не нужно добавлять реальные секреты и не нужно менять проект под платные сервисы.

## Что нужно сделать сейчас

Продолжи работу над репозиторием и подготовь его к ручному запуску.

Нужно сделать 5 вещей:

1. Добавить понятную пошаговую инструкцию запуска.
2. Добавить простой код события / invite code, чтобы случайный человек не мог легко писать в Google Таблицу.
3. Добавить красивую страницу или режим для показа итогов на гендер-пати.
4. Сделать настройку `APPS_SCRIPT_URL` понятнее и безопаснее для статического сайта.
5. Добавить ручной тест-чеклист перед отправкой ссылки гостям.

## 2. Invite code / event code

Добавь простую защиту от случайных отправок.

Требование:

- пользователь на сайте должен ввести короткий код события перед отправкой прогноза;
- Apps Script должен проверять этот код;
- код не должен быть Google-секретом или сервисным ключом;
- это именно простой invite/event code для вечеринки;
- если код неверный, прогноз не должен записываться в Google Таблицу.

Предпочтительный вариант:

```text
EVENT_CODE хранится в Script Properties Apps Script.
Frontend отправляет eventCode вместе с прогнозом.
Apps Script сравнивает payload.eventCode с EVENT_CODE.
```

Важно:

- не коммитить реальный код события;
- в `.env.example` и документации использовать placeholder;
- в `docs/security-notes.md` объяснить, что это не полноценная авторизация, а защита от случайного спама.

На сайте добавить поле:

```text
Код события
```

Валидация:

- код обязателен;
- лишние пробелы обрезать;
- при неверном коде показать понятное сообщение.

## 3. Экран итогов для гендер-пати

Добавь отдельную красивую страницу или отдельный режим для показа итогов.

Предпочтительный вариант:

```text
public/display.html
public/display.js
```

Страница должна показывать:

- текущие коэффициенты;
- сумму виртуальных ставок на мальчика;
- сумму виртуальных ставок на девочку;
- статус приёма прогнозов: открыт / закрыт;
- крупный красивый экран, который можно открыть на ноутбуке или телевизоре.

Не показывать:

- фамилии гостей;
- полный список участников;
- полный список прогнозов;
- служебные данные.

Если для display-страницы нужен только публичный `getState`, используй уже существующий endpoint.

Добавь ссылку на display-страницу из основной страницы или README.

## 4. Настройка APPS_SCRIPT_URL

Сейчас в `public/app.js` есть:

```js
const APPS_SCRIPT_URL = '';
```

Сделай так, чтобы настройка была понятнее.

Можно выбрать один из простых вариантов:

### Вариант A

Оставить константу в `public/app.js`, но:

- добавить заметный комментарий над ней;
- описать это в `docs/deploy.md`;
- объяснить, что пока URL пустой, сайт работает только в демо-режиме через `localStorage`.

### Вариант B

Вынести настройку в отдельный файл:

```text
public/config.js
```

Например:

```js
window.SITE_FOR_BABY_CONFIG = {
  APPS_SCRIPT_URL: '',
};
```

Если выбираешь вариант B, обнови `index.html`, `display.html` и документацию.

Приоритет — простота. Не добавляй сборщик, npm, Vite, Webpack, backend-хостинг или платные сервисы.

## 5. Ручной тест-чеклист

Создай файл:

```text
docs/manual-test-checklist.md
```

В чек-листе должны быть проверки:

### До публикации

- `.env` не закоммичен;
- реальных Google-ключей нет в коде;
- `SPREADSHEET_ID` хранится в Script Properties;
- `EVENT_CODE` хранится в Script Properties;
- Google Таблица не опубликована для гостей.

### Apps Script

- `setupSpreadsheet` создаёт листы;
- `getState` возвращает коэффициенты;
- `submitPrediction` записывает прогноз;
- неверный `eventCode` не записывает прогноз;
- `betting_open = false` запрещает отправку прогноза.

### Сайт

- главная страница открывается;
- коэффициенты загружаются;
- без имени отправить нельзя;
- без фамилии отправить нельзя;
- без кода события отправить нельзя;
- без выбора пола отправить нельзя;
- сумма должна быть больше 0;
- успешный прогноз появляется в Google Таблице;
- после прогноза коэффициенты меняются.

### Экран итогов

- display-страница открывается;
- показывает коэффициенты;
- показывает суммы по мальчику и девочке;
- не показывает персональные данные гостей.

## Что не делать

Не делай сейчас:

- Telegram-бота;
- онлайн-оплату;
- реальные деньги;
- платный хостинг;
- платную базу;
- покупку домена;
- сложную авторизацию;
- полноценную админку;
- npm/Vite/Webpack, если можно обойтись простым статическим сайтом;
- хранение Google-секретов во frontend.

## Безопасность

Проверь, что после изменений:

- Google Таблица не открывается гостям напрямую;
- frontend не содержит Google-ключей;
- Apps Script валидирует входные данные;
- Apps Script не возвращает полный список участников;
- ошибки не раскрывают stack trace;
- `.env` не коммитится;
- реальные значения `SPREADSHEET_ID`, `EVENT_CODE`, Web App URL не добавлены в репозиторий, если это не безопасный публичный placeholder.
