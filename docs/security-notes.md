# Безопасность MVP

## Что защищаем

- доступ к Google Таблице;
- настройки Apps Script;
- список участников и прогнозов;
- админские действия;
- пользовательский ввод.

## Основные правила

- Google Таблица не публикуется для гостей.
- Frontend не содержит Google-ключей.
- Frontend отправляет данные только в Apps Script.
- Apps Script валидирует все входные данные.
- Apps Script проверяет `EVENT_CODE` перед записью прогноза.
- Публичный endpoint не возвращает полный список прогнозов.
- Публичный endpoint может возвращать коэффициенты, статус приёма прогнозов и количество голосов.
- Публичный интерфейс не должен показывать фамилии гостей и список прогнозов.
- `.env` не коммитится.
- `.env.example` содержит только безопасные placeholder-значения.

## Event code

`EVENT_CODE` — это простой код события, а не полноценная авторизация.

Он нужен, чтобы случайный человек, который нашел ссылку, не мог легко записывать мусорные прогнозы в Google Таблицу.

Ограничения:

- код вводит сам гость;
- код не должен быть Google-ключом, паролем от аккаунта или сервисным секретом;
- реальное значение хранится в Script Properties Apps Script;
- реальное значение не коммитится в репозиторий;
- если код стал известен посторонним, его нужно заменить в Script Properties.

## Валидация

Проверять:

- имя обязательно;
- фамилия обязательна;
- код события обязателен;
- длина имени, фамилии и кода ограничена;
- `gender` только `boy` или `girl`;
- `amount` не меньше 100;
- `amount` не больше разумного лимита;
- прием прогнозов открыт.

## Публичное состояние сайта

Для публичного сайта достаточно отдавать:

```text
ok
bettingOpen
odds.boy
odds.girl
counts.boy
counts.girl
```

`counts` — это количество прогнозов по полу. Это не список гостей и не сумма виртуальных ставок.

Если frontend не нуждается в публичном отображении виртуальных сумм, лучше не показывать их в интерфейсе. Суммы остаются в закрытой Google Таблице для организатора и используются для расчёта коэффициентов.

## Одновременные отправки

В Google Apps Script используется `LockService`, чтобы два гостя не записали прогнозы одновременно с неправильным коэффициентом.

## Админские данные

На первом MVP организатор работает напрямую в Google Таблице. Публичная админка не нужна.

Если позже появится админский endpoint, для него нужен отдельный токен или другой способ ограничения доступа. Такой токен нельзя хранить в публичном frontend.
