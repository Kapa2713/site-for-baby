# site-for-baby

Проект для гендер-пати: небольшой онлайн-сервис, где гости делают игровые прогнозы на пол ребёнка — мальчик или девочка.

Важно: проект не принимает онлайн-платежи, не хранит реальные платёжные данные и не выплачивает реальные деньги. Это развлекательная механика для вечеринки. Денежные расчёты, если они вообще будут, проходят только офлайн между организаторами и гостями.

## Цель

Сделать простой MVP, который позволит:

1. Показать гостям текущие коэффициенты на мальчика и девочку.
2. Принять игровой прогноз от гостя.
3. Сохранять, кто, сколько виртуальных денег поставил и на какой вариант.
4. Автоматически пересчитывать коэффициенты после новых прогнозов.
5. Дать организаторам админ-экран со всеми прогнозами, докупками и общей статистикой.
6. На гендер-пати показать красивый итоговый экран.

## Формат продукта

Приоритет на старте — простой сайт.

Telegram-бот можно рассмотреть позже, если сайт окажется неудобным для гостей.

## Что входит в MVP

- публичная страница с текущими коэффициентами;
- форма участника: имя + выбор пола + сумма виртуальной ставки;
- виртуальный баланс участника или ручная фиксация суммы ставки;
- история прогнозов;
- пересчёт коэффициентов после каждого прогноза;
- админ-страница для организаторов;
- учёт докупок виртуальных денег;
- итоговая статистика: кто поставил, сколько, на какой вариант, сколько раз докупался;
- возможность закрыть приём прогнозов перед раскрытием пола.

## Что не входит в MVP

- онлайн-оплата;
- банковские карты;
- криптовалюта;
- реальные выигрыши;
- сложная авторизация;
- мобильное приложение;
- Telegram-бот как обязательная часть первого релиза.

## Базовая логика коэффициентов

Коэффициенты должны меняться похоже на букмекерскую механику: если больше виртуальных денег поставили на один вариант, коэффициент на него снижается, а на противоположный вариант растёт.

Предлагаемая простая формула для MVP:

```text
seed = 1000
boy_score = boy_total + seed
girl_score = girl_total + seed
total_score = boy_score + girl_score
boy_odds = total_score / boy_score
girl_odds = total_score / girl_score
```

На старте оба коэффициента равны `2.00`. Чем больше ставят на мальчика, тем ниже коэффициент на мальчика и выше коэффициент на девочку. И наоборот.

Коэффициент фиксируется в момент прогноза, чтобы потом можно было показать, по какому коэффициенту человек сделал прогноз.

## Роли

### Гость

Может:

- открыть сайт;
- ввести имя;
- посмотреть текущие коэффициенты;
- выбрать мальчика или девочку;
- указать сумму виртуальной ставки;
- отправить прогноз;
- увидеть, что прогноз принят.

### Организатор

Может:

- смотреть все прогнозы;
- смотреть суммы по мальчику и девочке;
- смотреть докупки;
- вручную добавлять докупку;
- закрывать приём прогнозов;
- видеть итоговую таблицу для вечеринки.

## Данные

Минимальные сущности:

- participant — участник;
- prediction — прогноз/ставка;
- top_up — докупка виртуальных денег;
- event_settings — настройки вечеринки;
- odds_snapshot — коэффициенты на момент ставки.

## Важные ограничения

- Не хранить реальные платёжные данные.
- Не использовать реальные деньги внутри приложения.
- Не обещать реальные выигрыши в интерфейсе.
- Не коммитить `.env` и секреты.
- Для настроек использовать `.env.example`.
- Не усложнять архитектуру без причины.
